# TraceEnemy（追従敵）仕様書

## 概要と設計意図

### 役割
- 追従敵のオブジェクト。
- 基本的な設計は通常敵と同じ、主に移動、攻撃などが違う。
- 移動・攻撃・ダメージ・描画など機能を自己管理する
### 設計意図
- Enemyクラスからの派生クラス
- 継承した共通処理（ダメージ処理・出現処理等）を利用しつつ、固有の移動処理・攻撃処理を記載する。
- 移動や攻撃などの処理を自己完結するようにし、他クラスに依存しない設計。
---

## 基本パラメータ（初期設定）
pop関数及びstart関数で設定される、自機の基本性能を記載します
### 初期位置
- 通常ステージとボスステージの2パターン
- 通常ステージ（NOMAL_STAGE） pop関数の引数で指定： 画面外上部2箇所
  - 左側（ENEMY_POP_LEFT_POSITION, ENEMY_POP_Y）（160,0）
  - 右側（ENEMY_POP_RIGHT_POSITION, ENEMY_POP_Y）（480,0）

- ボスステージ(BOSS_STAGE)  pop関数の引数で指定：画面外上部2箇所
  - 左側（SUMMON_LEFT_TRACE_ENEMY_X, SUMMON_ENEMY_Y）（50,0）
  - 右側（SUMMON_RIGHT_TRACE_ENEMY_X, SUMMON_ENEMY_Y）（630,0）

### 当たり判定サイズ
- 画像サイズ（64x48）と同じ。
- 常に一定。

### 移動速度
- （ENEMY_VECTOR_X）（1）X軸；1フレームで1ピクセル移動　
- （ENEMY_VECTOR_Y）（6.5）y軸：1フレームで6.5ピクセル移動
- 更新処理によって値は加減します。

- 追従速度定数 TRACE_ENEMY_TRACK_SPEED(4.0)
　- cos(),Sin()の値はー1から1の間で正規化されているため、そこへかけ合わせるスピード値。
　- 1フレームで4ピクセル移動

### 初期ライフ
通常ステージ（NOMAL_STAGE）：(ENEMY_MAX_HP：30)
ボスステージ（BOSS_STAGE）：(ENEMY_MAX_HP_BOSS_STAGE：60)

## 移動仕様

### 移動パターン
ステージによって2種類の移動パターンがあります
| ステージ| 移動内容|
| ---- | ----- |
| 通常ステージ（NOMAL_STAGE） | 画面外上部の左右いずれかから出現し、Y座標100まで降下する。その後、プレイヤーを追従して移動する。画面外へ到達する事がないので撃破されるまで継続する。      |
| ボスステージ（BOSS_STAGE）  | 画面外上部の左右いずれかから出現し、指定位置まで移動する。指定位置で一定時間停止した後、出現した方向へ向かって退場し、画面外へ到達すると消滅する。 |

### プレイヤー追従ロジック
- （プレイヤー座標 `playerX`、`playerY` を用いた追従計算の仕様）

### ボスステージ移動詳細
- 通常敵と同様。

| 項目  | 内容 |
| ---- | ---- |
| 目標座標 | （TRACE_ENEMY_SUMMON_POINT_LEFT_X, TRACE_ENEMY_SUMMON_POINT_LEFT_Y） = (190, 160) |
| 停止時間 | 出現から250フレーム（ENEMY_EXIT_START_FRAME：約4秒）|
| 退場方向 | 出現した側へ向かって移動 |
| 終了条件 | 画面外へ到達した時点で消滅  |
