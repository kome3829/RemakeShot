# BossEnemy（ボス敵）仕様書
## 概要と設計意図
### 役割
- ゲームの最終目標：ボスステージで待ち受けている最強の敵オブジェクト、ボス敵の撃破がゲームクリアの条件になっています。
- HP連動型の行動変化：雑魚敵とは異なり、残りHPに応じて「移動パターン」と「攻撃（弾幕）パターン」が段階的に変化します。
- UIとの連動：ボスステージでは、画面上部に専用の残りHPに連動したHPバーを表示し、プレイヤーに伝えます。
- 
### 設計意図
- 更新・描画・初期化を自身で管理する。他のクラスに依存しない設計。
- ボステージでのボスとの戦闘がゲームの主軸としています。この部分で楽しませるように考えてゲームを作成しています。
- ボス敵は基本的にボステージでの一騎打ちを想定した敵です
- HPの割合によってボスの状態を管理して、移動、攻撃のパターンを変化させています。
- **クラスの継承**
  - `Enemy` クラスの派生クラスとして実装し、ダメージ処理・出現処理等の共通処理を継承することで保守性を高める。

## 移動仕様
ボス敵の移動仕様は、残りHPの割合に応じて4つの行動フェーズに分かれており、フェーズ移行時には「画面中央への復帰処理」が挟まれます。

### 出現時の移動
- 降下移動：画面外上部中央（x=320,y=0）から出現し、規定の高度（`BOSS_DESCEND_LIMIT_Y`: 40）に達するまで垂直降下します。
- 減速処理: 降下中は `BOSS_ENEMY_SLOWDOWN_Y`（0.98）をy軸ベクトルに乗算して徐々に減速しながら定位置につきます

### HP割合に応じた移動パターン

| フェーズ | HP割合 | 移動内容 |
|:--------|:------:|:---------|
| 第1フェーズ | 100%～76% | **左右往復移動**：画面上部を水平に往復します。プレイヤーが狙いやすいよう、シンプルな軌道に設定されています。 |
| 第2フェーズ | 74%～51% | **4点巡回移動**：画面内の4つのポイント（左・下・右・上）を順番に巡回します。大きく位置を変えることでプレイヤーに移動を促します。 |
| 第3フェーズ | 49%～26% | **4点ワープ移動**：水平移動とワープを組み合わせた挙動です。フェードアウトして姿を消した後、別のポイントへ即座に出現し、水平移動を繰り返します。プレイヤーを翻弄します。 |
| 最終フェーズ | 24%～1% | **ワープ＆攻撃移動**：第3フェーズの4点ワープと中央での静止チャージ攻撃をループします。これまでのパターンを組み合わせた最も難易度が高いフェーズです。 |

### フェーズ移行処理（行動の切り替え）
HPが特定の値（75%, 50%, 25%）に達すると、次の行動へ移るための準備状態に入ります。

**無敵化**
- 切り替え中は`mUnbeatable` が true になり、ダメージを受け付けないようにします。

**中央位置へ復帰**
- 次の行動に開始するため、画面中央上部へと移動します

| 項目 | 値 | 使用定数 |
|:-----:|:----:|:----------:|
|中央位置x座標 | 320 |  `SCREEN_CENTER_X` |
|中央位置y座標 | 100 |  `BOSS_CENTER_POSITION_Y` |

**HPの強制減算**
- 移行処理を完了させるため、内部的に `START_MOVE_DAMAGE`（30）を減算して次のHP範囲へ押し込みます

### ワープ移動演出
第三フェーズ以降で使用されるワープ移動時の演出は以下の流れで行います。

1. 消失演出とエフェクト再生
   - `mWarp` フラグを立て、透過度（`mAlpha`）を下げフェードアウトしながら、上空へ加速上昇します
   - ワープエフェクトを再生します。

2. 座標更新
   - 完全に透明になったタイミングで、目的地へ座標をテレポートさせます

3. 再出現
   - 目的地で `mAlpha` を最大値に戻し、再び姿を現します
   
### 移動補足
ボスの移動では、ベクトル計算と誤差判定を組み合わせています。

| 項目 | 内容 |
|:-----|:-----|
| ベクトル移動 | `atan2()` で目標方向を算出し、`cos()`・`sin()` により速度ベクトルを計算する。 |
| 到達判定 | `POSITION_TOLERANCE` (3) の範囲内に入った時点で目標地点へ到達したと判定する。 |
| 目的 | 小数点誤差を考慮して、安定した移動を実現する。 |

## 攻撃仕様
ボスの残りHPに連動して4段階（フェーズ）で攻撃が激化します。

### 攻撃設計意図
- **段階的な難易度の上昇**
   - ボスの残りHPに連動して攻撃密度と種類を増やし、プレイヤーに最後まで緊張感を与える

### 使用する弾

| 弾種 | 特徴 |
|:-----|:-----|
| ミニ敵弾(黄色) | 小型の弾。高頻度で発射され、主に円状の弾幕（環状弾）として使用される。 |
| 通常敵弾 | 標準サイズの弾。発射後に徐々に減速し、画面内に滞留して回避を困難にする。 | 

### フェーズ別攻撃パターン

| フェーズ | HP割合 | 使用弾種 | 攻撃パターン | 発射数 / 間隔（定数） |
|:---------|:------:|:---------|:-------------|:----------------------|
| 第1フェーズ | 100% ～ 75% | ミニ敵弾 | **円状弾幕（基本）**：ボスの中心から全周囲へ均等に弾を放つ基本パターン。 | ・12方向同時発射（30度間隔）<br>・30フレーム間隔 |
| 第2フェーズ | 74% ～ 50% | ミニ敵弾 | **連射円状弾幕**：第1フェーズの弾幕を短い間隔で連続発射し、弾幕の密度を高める。 | ・12方向同時発射<br>・4回連続発射<br>・連射間隔：4フレーム |
| 第3フェーズ | 49% ～ 25% | ミニ敵弾<br>通常敵弾 | **複合弾幕**：連射ミニ弾に加え、発射後に減速する通常弾を混ぜて回避を困難にする。 | ・第2フェーズの攻撃に加えて<br>・通常弾：30方向同時発射（12度間隔）<br>・通常弾発射間隔：120フレーム |
| 最終フェーズ | 24% ～ 0% | ミニ敵弾<br>通常敵弾<br>（雑魚召喚） | **狂乱弾幕 ＆ 雑魚連携**：ミニ弾の発射角度を毎フレームずらして回転弾幕を形成。さらに三種の雑魚敵を召喚し、波状攻撃を仕掛ける。 | ・ミニ弾：発射角度を一定間隔で変更<br>・通常弾：30方向同時発射（第3フェーズ継続）<br>・三種の雑魚敵を随時召喚 |

##  ダメージ処理
基本雑魚敵と同じ処理ですが撃破時の処理等の違いとフェーズ切り替え時の強制HP減算があります。
### 実行トリガー
- 外部（弾）の当たり判定処理によってダメージフラグ `mTakeDamage` が true に設定された場合にのみ処理が実行
- 毎フレームの更新処理（action 関数）内で呼び出し


### 無敵・クールダウン判定
- 次の条件の場合ダメージ処理は行われない
  - 無敵フラグ（`mInvincible`）が true
  - ダメージ表示中（`mDamageCoolDown`）がtrue

### 内部処理
 - スコア加算: ヒットスコア（`HIT_SCORE`:300）が現在のスコアに加算
 - HP減算: 現在のHP（`mHitPoint`）からダメージ量（`ENEMY_TAKE_DAMAGE`:5）減算
 - クールダウン開始: ダメージフラグ `mDamageCoolDown` を true

### 表示処理
 - ダメージ画像の表示: `mDamageCoolDown` が true の間は、ダメージ用の画像(白色の色違い画像)が描画
 - 表示時間: ダメージ画像の表示は `DAMAGE_COUNT_LIMIT`（2フレーム）の間のみ。この値を超えると通常状態へ戻る。
 - 被弾時用更新処理：ダメージ中は `mDamageDisplayCount` を毎フレームカウントし、規定フレーム（2フレーム）を超えた瞬間に、カウントをリセット
### 撃破判定
- ダメージ処理の直後に、HPの状態を確認して撃破の判定が行われます。
- 判定条件：敵のHP（mHitPoint）が 0以下 になった際に撃破
### 撃破処理
撃破判定が成立すると、以下の処理を実行します。
- フラグ更新: 撃破フラグ（mDefeat）を true にし、ボス撃破演出へ移行します。

## 表示(描画仕様)
4つの表示状態があり、状態の応じた画像を表示しています。
- 実行条件: 有効状態（`mActive == true`）である場合のみ描画処理を実行
- 描画座標: 中心座標（`mX, mY`）を基準に描画します。画像描画関数は左上座標を基準とするため、画像の幅・高さの半分（`mWidth / 2、mHeight / 2`）を差し引いた座標を描画位置として計算します。
### 表示要素

| 表示状態 | 表示画像 | 説明 |
|:-----:|:----:|:----------:|
|通常時 | 通常ボス敵画像 |  ボス敵が有効状態でダメージやワープのフラグが経っていない場合に表示 |
|被弾時 | ダメージ用ボス敵画像 |  ダメージフラグ（`mDamageCoolDown`）が true の間で表示 |
|ワープ演出中 | ダメージ用ボス敵画像 |  ワープフラグ（`mWarp`）が true の間で表示、透過度の反映がされ、フェードアウトさせています。 |
|撃破時 | 通常ボス敵画像 | 撃破フラグ（`mDefeat`）が true の間で表示、透過度の反映がされ、フェードアウトさせています。 |
- ボスステージでのみ、画面上部に専用のHPバーが表示されます。ゲームシーンで描画処理を行っています。
- [ゲームシーン : GameScene_Specification.md](GameScene_Specification.md)

### 描画補足
#### 1. レイヤー順序（描画の優先順位）


| レイヤー | 描画対象 | 詳細 |
|:-------:|:---------|:-----|
| 1（最背面） | 背景動画 | 背景なので最背面になります |
| 2 | **ボス敵** | 背景の上に描画します。雑魚敵と優先順位です。 |
| 3 | 弾（敵弾・自機弾） | ボス敵より手前に描画します。ゲームプレイ上、弾の視認性を優先しています。 |
| 4（最前面） | UI（ボスHPバー・スコアなど） | キャラクターや弾の後に描画し、常に情報が隠れないようにします。 |

---

#### 2. 透過処理とブレンドモードの制御

ワープ演出中や撃破時のフェードアウト演出では、アルファブレンドを使用します。

| 項目 | 内容 |
|:-----|:-----|
| ブレンドモードの変更 | 透過度（`mAlpha`）を反映するため、描画直前に `SetDrawBlendMode(DX_BLENDMODE_ALPHA, mAlpha)` を実行します。 |
| ブレンドモードのリセット | 描画終了後に `SetDrawBlendMode(DX_BLENDMODE_NOBLEND, 0)` を実行し、他オブジェクトへの影響を防ぎます。 |

---

#### 透過度の制御

| 使用定数 | 値 | 内容 |
|:---------|:--:|:-----|
| `ALPHA_MAX` | 255 | 通常時の透過度 |
| `FADE_ALPHA_STEP` | 25 | フェードアウト時に毎フレーム減算する値 |

## 補足・テクニカル事項

###  フェーズ移行の強制HP減算

HP割合によるフェーズ移行を確実に行うため、準備処理完了時に内部でHPを減算します。

| 項目 | 内容 |
|:-----|:-----|
| 使用定数 | `START_MOVE_DAMAGE` (30) |
| 実行タイミング | `settingNext...()` 関数の完了時 |
| 目的 | HP割合による分岐条件から現在のフェーズを強制的に脱出させ、次のフェーズへ確実に移行する。 |

###  雑魚召喚

ボスとゲームシーンの責務を分離し、クラス間の結合度を低減しています。

| 項目 | 内容 |
|:-----|:-----|
| 設計方針 | 雑魚敵の生成処理はゲームシーン側へ委譲する。 |
| ボス側の処理 | `mSummonEnemy` フラグを設定するのみ。 |
| ゲームシーン側 | フラグを検知し、雑魚敵の `pop()` を実行する。 |
| 目的 | ボスクラスが雑魚敵マネージャーへ直接依存しない設計にする。 |
