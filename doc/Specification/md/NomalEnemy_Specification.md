# Enemy（通常敵）仕様書
## 概要と設計意図
### 役割
通常敵のオブジェクト。シンプルな動き、攻撃を行う。
### 設計意図
更新・描画・初期化を自身で管理する。他のクラスに依存しない設計。
敵の基本形となり、他の敵の作成の土台とする。

## 出現処理（pop）

敵オブジェクトを有効化し、初期状態を設定する処理です。  

### 実行内容

- **有効化**
  - `mActive` を `true` に設定し、更新（`action()`）および描画（`draw()`）の対象とする。
- **撃破状態のリセット**
  - `mDefeat` を `false` に設定する。
- **座標・個体番号の設定**
  - 引数で指定された座標（`popPositionX`、`popPositionY`）を設定する。
  - 個体判別番号（`mNumber`）を設定する。

### ステージ別の初期化

| 設定項目 | 通常ステージ（`BossStage = false`） | ボスステージ（`BossStage = true`） |
|----------|---------------------------------------|--------------------------------------|
| 初期ライフ | `ENEMY_MAX_HP`（30） | `ENEMY_MAX_HP_BOSS_STAGE`（60） |
| 開始カウント | `0`（コンストラクタ等の初期値） | `BOSS_STAGE_START_COUNT`（10） |
| 対象フラグ | `mBossStage = false` | `mBossStage = true` |

### 初期化の補足

ボスステージでは、召喚直後からボスの攻撃パターンと同期させる必要があるため、`mMoveCount` および `mShotCount` を `BOSS_STAGE_START_COUNT`（10）から開始することで、初期動作のタイミングを調整している。

## 移動仕様
### 移動パターン
ステージによって2種類の移動パターンがあります
| ステージ| 移動内容|
| ---- | ----- |
| 通常ステージ（NOMAL_STAGE） | 画面外上部の左右いずれかから出現し、Y座標100まで降下する。その後、出現した位置とは反対方向へ水平移動し、画面外へ到達すると消滅する。      |
| ボスステージ（BOSS_STAGE）  | 画面外上部の左右いずれかから出現し、指定位置まで移動する。指定位置で一定時間停止した後、出現した方向へ向かって退場し、画面外へ到達すると消滅する。 |

### ボスステージ移動詳細
| 項目  | 内容 |
| ---- | ---- |
| 目標座標 | （ENEMY_SUMMON_POINT_LEFT_X, ENEMY_SUMMON_POINT_LEFT_Y） = (130, 160) |
| 退場開始時間 | 出現から250フレーム（ENEMY_EXIT_START_FRAME：約4秒）|
| 退場方向 | 出現した側へ向かって移動 |
| 終了条件 | 画面外へ到達した時点で消滅  |
#### 指定位置への移動ロジック
- 指定位置への移動には atan2() を用いて移動角度を算出する
- 算出した角度から cos()・sin() を利用して速度ベクトルを求める
- 指定位置と現在の座標のずれを計算し、一定値(`ENEMY_POSITION_DIFFERENCE_RANGE`)内に収まると停止する。


#### 退場移動
- 出現からの経過フレーム数（mMoveCount）が 250フレーム（ENEMY_EXIT_START_FRAME）に達した瞬間に、退場移動へ移行する
- 退場開始時に速度ベクトルが設定されます

| 出現位置 | 退場時速度(x,y) | 退場方向 |
| ------ | ------ | ------ |
|左側（SUMMON_LEFT）|(-ENEMY_EXIT_SPD,-ENEMY_EXIT_SPD)=(-1,-1)|左上方向に脱出|
|右側（SUMMON_RIGHT）|(ENEMY_EXIT_SPD,-ENEMY_EXIT_SPD)=(1,-1)|右上方向に脱出|
- 退場中ではy軸速度ベクトルに加速処理が行われます。毎フレーム（ENEMY_SPEEDUP_Y：1.08）乗算されます。
- 退場中は攻撃を停止させるため、mShotCount を常に 0 に固定

## 攻撃仕様
- 攻撃パターンは1種類のみ。
- 通常ステージ・ボスステージ共通で同じ攻撃パターンを使用する。
### 攻撃設計意図
- シンプルで比較的よけやすい掃射パターンにする。
- 発射カウントをそのまま発射角度に利用するのは計算処理を省くため。
### 弾発射処理
- 弾発射処理の概要
  - 160フレームを1サイクルとして動作する。
  - 発射開始フレームから終了フレームまで、10フレームごとに計5発発射する。

#### 弾発射処理実装仕様

| 項目  | 内容  |
| ------------ | ----|
| 使用する弾        | 通常敵弾（赤い丸い弾）  |
| 攻撃サイクル       | 160フレーム（mShotCount > 160 になった時点でカウントを0にリセット） |
| 発射開始フレームカウント | 60 (ENEMY_SHOT_START_FRAME) |
| 発射終了フレームカウント | 100 (ENEMY_SHOT_STOP_FRAME)  |
| 発射レート        | 10フレームごとに1発発射  |
| 実際の発射フレーム    | 60、70、80、90、100 |
| 弾生成位置        | 敵オブジェクトの中心座標から生成   |

#### 発射角度の仕様
- 発射時のカウント値をそのまま角度として利用する。10度ずつ角度をずらす。
- 実際の発射角度:60, 70, 80, 90, 100（敵から見て左から右へ掃射する）
#### 発射演出
- 弾の生成と同時に発射SEを再生する。
- 同時に発射エフェクトを生成する。
### 移動処理との関係
- 移動処理と攻撃処理はそれぞれ独立したカウントで管理する。
- 移動状態に関係なく、攻撃カウントに従って発射処理を行う。
- ボスステージでは退場中、発射カウントを0に固定することで攻撃を停止する。

## ダメージ処理
### 実行トリガー
- 外部（弾）の当たり判定処理によってダメージフラグ mTakeDamage が true に設定された場合にのみ処理が実行
- 毎フレームの更新処理（action 関数）内で呼び出し
### 無敵・クールダウン判定
- 次の条件の場合ダメージ処理は行われない
  - 無敵フラグ（mInvincible）が true
  - ダメージ表示中（mDamageCoolDown）がtrue
### 内部処理
 - スコア加算: ヒットスコア（HIT_SCORE:300）が現在のスコアに加算
 - HP減算: 現在のHP（mHitPoint）からダメージ量（ENEMY_TAKE_DAMAGE:5）減算
 - クールダウン開始: ダメージフラグ mIsDamageCoolDown を true
### 表示処理
 - ヒットエフェクト: 敵の中心座標（mX, mY）にヒットエフェクト（HIT_EF）を表示。
 - ダメージ画像の表示: mDamageCoolDown が true の間は、ダメージ用の画像(青色の色違い画像)が描画
 - 表示時間: ダメージ画像の表示は DAMAGE_COUNT_LIMIT（2フレーム）の間のみ。この値を超えると通常状態へ戻る。
 - 被弾時用更新処理：ダメージ中は mDamageDisplayCount を毎フレームカウントアップし、規定フレーム（2フレーム）を超えた瞬間に、カウントをリセット
### 撃破判定
- ダメージ処理の直後に、HPの状態を確認して撃破の判定が行われる。
- 判定条件：敵のHP（mHitPoint）が 0以下 になった際に撃破とする
### 撃破処理
撃破判定が成立すると、以下の処理を実行する
- フラグ更新: 撃破フラグ（mDefeat）を true にし、生存フラグ（mActive）を false にしてオブジェクトを無効化
- 撃破スコア加算: 敵撃破スコア（ENEMY_SCORE:1000）が加算
- 撃破演出：撃破SEと撃破用爆発エフェクト(EXPLOSION_E)を再生
- アイテムドロップ: アイテム管理クラス（ItemObjectManager）を通じて、以下の3種類のアイテムをドロップ
  - 経験値アイテム（OBJECT_EXP）:固定で６つドロップ
  - 回復アイテム（OBJECT_LIFE）:乱数による確率で10％でドロップ
  - 無敵アイテム（OBJECT_STAR）:乱数による確率で30％でドロップ

## 表示(描画仕様)
- 実行条件: 敵が有効状態（mActive == true）である場合のみ描画処理を実行
- 描画座標: 敵の中心座標（mX, mY）を基準とする。画像を描画する際は、画像の幅・高さの半分（mWidth / 2, mHeight / 2）を差し引いた座標を計算して描画
- ダメージの更新ロジックはダメージ処理項目で記載。

### 表示要素
- 通常画像：ダメージ以外の時に表示する敵画像（赤色）
- ダメージ画像：被弾時用色違いの敵画像（青色）。被弾後、mDamageCoolDown が true の間表示
### 描画補足（描画順序）
- レイヤー順: GameMainScene の draw 関数において、背景動画の後に描画されること。また、敵の後に弾を描画すること。ゲームプレイする上で弾の方が重要度が高いため。

## 補足・テクニカル事項
### 引数管理の構造体化
action 関数の引数が肥大化するのを防ぐため、必要なマネージャークラスのポインタをまとめた EnemyActionData 構造体を使用している。
### オーバーフロー対策
移動制御に用いる mMoveCount が、型（int）の最大値付近である MOVE_COUNT_LIMIT (2,147,483,646) に達した場合、0にリセットしてオーバーフローを防止する
### メモリ管理
クラス内メンバは基本変数（int, bool等）のみであるため、デストラクタでの個別 delete は不要（自動解放される）
