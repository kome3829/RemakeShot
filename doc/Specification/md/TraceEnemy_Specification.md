# TraceEnemy（追従敵）仕様書

## 概要と設計意図

### 役割
プレイヤーを追跡する挙動を持つ敵オブジェクト。通常敵（Enemy クラス）の設計を基に、移動及び攻撃機能を独自のものに拡張している。
  - プレイヤー追尾挙動:　出現後、一定高度（y=100 / ENEMY_DESCEND_LIMIT_Y）まで降下したのち、プレイヤーの現在座標を追いかけるように移動
  - 攻撃機能: 　4方向への同時弾発射に加え、角度を変えながらの連続掃射
### 設計意図
- クラスの継承：Enemyクラスからの派生クラス。継承した共通処理であるダメージ処理（takeDamage）・出現処理（pop）等を利用して、効率と保守性を高める。
- 自律的な管理: 移動や攻撃などの処理を自己完結するようにし、必要なクラスとのみ連携する設計。
- プレイヤーの座標を基に移動する敵を追加することでゲームの難易度を変化させる。

## 出現処理（pop）

敵オブジェクトを有効化し、初期状態を設定する処理です。  
親クラス `Enemy` の `pop()` 関数を共通の出現処理として使用します。

### 実行内容

- **有効化**
  - `mIsActive` を `true` に設定し、更新（`action()`）および描画（`draw()`）の対象とする。
- **撃破状態のリセット**
  - `mIsDefeat` を `false` に設定する。
- **座標・個体番号の設定**
  - 引数で指定された座標（`popPositionX`、`popPositionY`）を設定する。
  - 個体判別番号（`mNumber`）を設定する。

### ステージ別の初期化

| 設定項目 | 通常ステージ（`isBossStage = false`） | ボスステージ（`isBossStage = true`） |
|----------|---------------------------------------|--------------------------------------|
| 初期ライフ | `ENEMY_MAX_HP`（30） | `ENEMY_MAX_HP_BOSS_STAGE`（60） |
| 開始カウント | `0`（コンストラクタ等の初期値） | `BOSS_STAGE_START_COUNT`（10） |
| 対象フラグ | `mIsBossStage = false` | `mIsBossStage = true` |

### 初期化の補足

ボスステージでは、召喚直後からボスの攻撃パターンと同期させる必要があるため、`mMoveCount` および `mShotCount` を `BOSS_STAGE_START_COUNT`（10）から開始することで、初期動作のタイミングを調整している。

## 移動仕様

### 移動パターン

ステージに応じて、以下の内容で移動する。

| ステージ | フェーズ | 移動内容 |
|----------|----------|----------|
| 通常ステージ | 1. 降下 | 画面外から出現し、規定高度（`ENEMY_DESCEND_LIMIT_Y`：`100`）まで直進降下する。 |
| 通常ステージ | 2. 追従 | 規定高度到達後、撃破されるまでプレイヤーを常に追従する。撃破されるまでプレイヤーを追従し続ける。 |
| ボスステージ | 1. 登場 | 召喚位置から目標座標まで最短距離で移動する。 |
| ボスステージ | 2. 停止 | 指定位置で一定時間経過まで（`ENEMY_EXIT_START_FRAME`：`250f`）停止し、その間に攻撃を行う。 |
| ボスステージ | 3. 退場 | 停止時間終了後、出現方向へ加速しながら退場する。画面外へ到達すると消滅する。 |

### プレイヤー追従移動詳細
-  自身の座標 (mX, mY) とプレイヤーの現在座標から、atan2() を用いて目標への角度を算出します
- 算出した角度を cos()（X軸）および sin()（Y軸）に適用し、移動方向を正規化された速度ベクトルを求めます。
- 速度ベクトルを利用して、移動を行います。
- ベクトル算出時には追従移動速度定数TRACE_ENEMY_TRACK_SPEED (4.0)を掛け合わせて毎フレームの移動量を決めています。
- 自身の座標 (mX, mY) に算出した速度ベクトル（mVectorX,mVectorY）に加算して位置を更新しています。

### ボスステージ移動詳細
- 通常敵と同様なパターン。出現位置と目標位置が追従敵用に設定されています。

| 項目  | 内容 |
| ---- | ---- |
| 目標座標 | （TRACE_ENEMY_SUMMON_POINT_LEFT_X, TRACE_ENEMY_SUMMON_POINT_LEFT_Y） = (190, 160) |
| 退場開始時間 | 出現から250フレーム（ENEMY_EXIT_START_FRAME：約4秒）|
| 退場方向 | 出現した側へ向かって移動 |
| 終了条件 | 画面外へ到達した時点で消滅  |

#### 指定位置への移動ロジック
- 指定位置への移動には atan2() を用いて移動角度を算出する
- 算出した角度から cos()・sin() を利用して速度ベクトルを求める
- 指定位置と現在の座標のずれを計算し、一定値(`ENEMY_POSITION_DIFFERENCE_RANGE`)内に収まると停止する。


#### 退場移動
- 出現からの経過フレーム数（mMoveCount）が 250フレーム（ENEMY_EXIT_START_FRAME）に達した瞬間に、退場移動へ移行する
- 退場開始時に速度ベクトルが設定されます

| 出現位置 | 退場時速度(x,y) | 退場方向 |
| ------ | ------ | ------ |
|左側（SUMMON_LEFT）|(-ENEMY_EXIT_SPD,-ENEMY_EXIT_SPD)=(-1,-1)|左上方向に脱出|
|右側（SUMMON_RIGHT）|(ENEMY_EXIT_SPD,-ENEMY_EXIT_SPD)=(1,-1)|右上方向に脱出|
- 退場中ではy軸速度ベクトルに加速処理が行われます。毎フレーム（ENEMY_SPEEDUP_Y：1.08）乗算されます。
- 退場中は攻撃を停止させるため、mShotCount を常に 0 に固定

## 攻撃仕様
- 攻撃パターンは１種類のみ。
- 通常ステージ・ボスステージ共通で同じ攻撃パターンを使用する。
### 攻撃設計意図
- 攻撃は4方向同時発射と5発連続掃射を組み合わせたものにしています。
- 四方向同時にすることで、追従移動に合わせて追従敵がどの方向にいてもをプレイヤーへ圧をかけられるよう
してます。掃射にすることで攻撃範囲を広くしています。
- 発射カウントを発射角度に利用するのは計算処理を省くため。
- 
### 弾発射処理
- 弾発射処理の概要
  - 180フレームを1サイクルとして動作する。
  - 発射開始フレームから終了フレームまで、10フレームごとに計5発発射する。
  - 4方向90度毎に角度を変えて、同時に発射する。

### 弾発射処理実装仕様
| 項目  | 内容  |
| ------------ | ---- |
| 使用する弾        | 通常敵弾（赤い丸い弾）  |
| 攻撃サイクル      | （TRACE_ENEMY_SHOT_RESET_FRAME）（180）：mShotCount > 180 になった時点でカウントを0にリセット |
| 発射開始フレームカウント |（TRACE_ENEMY_SHOT_START_FRAME） （60）|
| 発射終了フレームカウント |（TRACE_ENEMY_SHOT_END_FRAME）（100） |
| 発射レート        |（TRACE_ENEMY_SHOT_INTERVAL_FRAME）（10）：10フレームごとに1発発射  |
| 実際の発射フレーム    | 60、70、80、90、100 |
| 弾生成位置        | 敵オブジェクトの中心座標から生成   |
| 同時発射方向        | （TRACE_ENEMY_SHOT_WAY_COUNT）（4）　:　4方向に同時に発射 |

### 発射角度
- 発射角度は。4方向へ同時発射とカウント値を利用した角度調整を組み合わせています。
- 同時発射用のループカウンタ（0～3）に、基準角度 90度（TRACE_ENEMY_SHOT_BASE_ANGLE）を掛け合わせることで、十字方向（360度を4分割）の射線方向へ調整してます。
- 各斜線方向の基準角度に発射カウント値を加算することで10度ずつ角度を変更しています。
- 結果4方向に10度ずつ角度が時計周りに変化します。

#### 発射演出
- 弾の生成と同時に発射SEを再生する。
- 同時に発射エフェクトを生成する。

## 表示(描画仕様)
- 実行条件: 敵が有効状態（mIsActive == true）である場合のみ描画処理を実行
- 描画座標: 敵の中心座標（mX, mY）を基準とする。画像を描画する際は、画像の幅・高さの半分（mWidth / 2, mHeight / 2）を差し引いた座標を計算して描画
- ダメージ画像の更新ロジックはダメージ処理項目で記載。

### 表示要素
- 通常画像：ダメージ以外の時に表示する敵画像（黄色）
- ダメージ画像：被弾時用色違いの敵画像（青色）。被弾後、mIsDamageCoolDown が true の間表示
### 描画補足（描画順序）
- レイヤー順: GameMainScene の draw 関数において、背景動画の後に描画されること。また、敵の後に弾を描画すること。ゲームプレイする上で弾の方が重要度が高いため。

## 補足・テクニカル事項
### 引数管理の構造体化
action 関数の引数が肥大化するのを防ぐため、必要なマネージャークラスのポインタをまとめた EnemyActionData 構造体を使用している。
### クラス継承
- Enemyクラスを継承した派生クラスになります。Enemyクラスから継承したメンバ変数のみを利用しています。
- takeDamage()（ダメージ処理）・pop()（出現処理）は親クラスのものをそのまま利用しており、重複を避けています。

### オーバーフロー対策
移動制御に用いる mMoveCount が、型（int）の最大値付近である MOVE_COUNT_LIMIT (2,147,483,646) に達した場合、0にリセットしてオーバーフローを防止する
### メモリ管理

クラス内メンバは基本変数（int, bool等）のみであるため、デストラクタでの個別 delete は不要（自動解放される）
