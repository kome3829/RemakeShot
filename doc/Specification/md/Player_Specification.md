## 概要と設計意図
### 役割
プレイヤーが操作する自機オブジェクト
移動・攻撃・ダメージ・アイテム取得・描画など、自機に関する機能を管理する
### 設計意図
更新・描画・衝突判定後の影響（アイテム取得など）を自身で管理する。他のクラスに依存しない設計。

## 基本パラメータ（初期設定）
コンストラクタおよび start 関数で初期化される、自機の基本性能を記載します
### 初期位置
画面中央（SCREEN_CENTER_X, SCREEN_CENTER_Y）（320,240）
### 当たり判定サイズ
- 縦横10px(PLAYER_HIT_RANGE)の四角形。
- 画像サイズ（64x48）よりも小さく設定し、回避しやすくしている
- 表示自機画像の中央にある点が当たり判定の目安
### 移動速度
- 2段階
- 通常時（PLAYER_SPEED）：1フレームで5ピクセル移動
- 低速移動時（PLAYER_LOW_SPEED）：1フレームで2.5ピクセル移動
### 初期ライフ
3つ：(MAX_LIFE)
初期状態で3つのHPを持っています。プレイヤー自機の下部のHPアイコンで確認できます。
## 操作仕様（入力と挙動）
### 移動操作
| キー | 動作 |
|------|------|
| ↑ ↓ ← → | プレイヤー移動 |
| Shift | 低速移動 |

- 上下左右キーによる移動
- 画面外への移動制限（画像サイズを考慮したクランプ処理）
- Shiftキー押下による低速移動（精密な回避用）
### 操作性の追求
キー入力の分岐をさせず同時押しを可能にすることで、直感的な斜め移動を実現している

### 攻撃操作
| キー | 動作 |
|------|------|
| Space | ショット |
- Spaceキーによるショット発射
- 発射間隔の制御（5フレーム間隔 PLAYER_NOMAL_SHOT_INTERVAL）

## 戦闘仕様
### ダメージ処理
- 被弾時のライフ減算と、一定時間（40フレーム PLAYER_DAMAGE_RESET_FRAME）のダメージ無効化（クールダウン）
- ダメージ時の専用画像表示演出
### アイテム取得
ゲームプレイの補助として敵を倒すとアイテムがドロップします。ドロップしたアイテムは近づくことで取得できます。
- プレイヤーはアイテムを取得して様々な効果を受けることができます。
- [アイテム : Item_Specification.md](Item_Specification.md)

### ライフ回復
回復アイテムを取得するとプレイヤーのライフを回復することができます。
   - 1つ獲得することでプレイヤーの現在ライフ（`mLife`）が 1つ増加します
   - ライフがすでに最大値（`MAX_LIFE`: 3）に達している場合は、それ以上増えません

### 無敵状態
無敵フラグ（`mUnbeatable`）が trueの時、敵の攻撃によるダメージを一切受けなくなります。
制限時間があり、切れると終了します。

#### トリガー
- 無敵アイテム取得時に、無敵フラグ（`mUnbeatable`）が trueになり、無敵状態へ入ります。
- 無敵状態時に取得した場合、制限時間を最大値へリセットします。

|項目|値|使用定数|
|---|---|---|
|制限時間|180F|`PLAYER_UNBEATABLE_MAX_FRAME`|
#### 無敵時の表示
- 無敵ゲージの表示
   - 自機下部に残り時間を表すオレンジ色のゲージを表示します。
- 表示画像
  - 自機の画像を黄色の色違いの画像を表示します。
- 終了予告演出
   - 無敵終了の60フレーム前から自機画像を通常時と無敵時を切り替え点滅表示させます。
### 撃破判定
ライフが0になった際に撃破フラグ（mIsDefeat）をfalseにする。プレイヤー撃破がゲームシーンの終了条件の１つとなっている。

### 発射パターン
ショットパワーに応じて、
発射する弾種および発射間隔が変化する
### プレイヤー発射弾表
| 弾種 | 発射数 | 発射間隔（フレーム） | 特徴 |
|------|:------:|:-------------------:|------|
| 通常弾 | 2 | 10フレーム（実質） | スペースキー押下で正面に発射する基本の攻撃。 |
| ミサイル弾 | 2 | 15フレーム（実質） | 斜め方向に発射し、左右に拡散する。 |
| スペシャル弾 | 5 | 25フレーム（実質） | 5発の弾を扇状に拡散して発射する特殊攻撃。 |

### ショットパワーによるパターン変化
ショットパワーは経験値アイテムの取得により加算されていきます。1つ獲得すると(`SHOT_POWER_INCREASE_AMONT`：30)増加します。
| ショットパワー | 発射内容  |
|------|------|
| 0～1499 |通常弾を実質10フレーム間隔で発射 |
| 1500～2999 | 通常弾を実質5フレーム間隔で発射 |
| 3000～5999 | 通常弾を実質5フレーム間隔で発射。加えて、左右2発のミサイル弾を実質15フレーム間隔で発射 |
| 6000～ | 通常弾を実質5フレーム間隔で発射。加えて、左右2発のミサイル弾を実質15フレーム間隔で発射し、さらに5方向のスペシャル弾を実質25フレーム間隔で5連続発射する |


## 表示(描画仕様)
### 表示要素

| 表示要素 | 内容 |
|----------|------|
| 通常画像 | ダメージ状態・無敵状態以外で表示するプレイヤー画像（青色）。 |
| ダメージ画像 | ダメージを受けた際に表示するプレイヤー画像（赤色）。 |
| 無敵画像 | 無敵状態の間に表示するプレイヤー画像（黄色）。 |
| 残機アイコン | プレイヤーの残りHPを表す青色のハート形アイコン。現在のHPに応じた数を表示する。 |
| 無敵時間ゲージ | 残り無敵時間を表示するゲージ。プレイヤー画像の幅（64px）を最大値とし、時間経過に応じて右から左へ減少する。 |

## 補足・テクニカル事項
### テスト用機能
TEST_STAGE における攻撃力の底上げ（6100）や常時無敵フラグの付与仕様
### メモリ管理
クラス内メンバは基本変数（int, bool等）のみであるため、デストラクタでの個別 delete は不要（自動解放される）
