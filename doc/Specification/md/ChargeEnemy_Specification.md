# ChargeEnemy（チャージ敵）仕様書

##  概要と設計意図

### 役割
チャージ（溜め）をしてプレイヤー狙い撃ち攻撃を行う敵オブジェクト。通常敵（Enemy クラス）の設計を基に、移動及び攻撃機能を独自のものに拡張している。
- **移動と強力な攻撃**
  - 通常ステージでは放物線を描く特殊な移動を行う。
  - ボスステージではボスの攻撃に合わせてワープや円運動を交えたトリッキーな動きを仕掛ける。
  - チャージ後にプレイヤー狙い撃ち攻撃を行う。
- **自己管理**
  - 移動・攻撃・ダメージ・描画などの機能をクラス内で完結して管理する。

### 設計意図

- **クラスの継承**
  - `Enemy` クラスの派生クラスとして実装し、ダメージ処理・出現処理等の共通処理を継承することで保守性を高める。
- **ボスとの連携**
  - ボスステージでは他の雑魚敵とは異なり、ボスの攻撃パターンに合わせた専用移動・攻撃を持つ。
    - 召喚位置への移動
    - チャージ
    - ワープ
    - 円運動＆３射線のプレイヤー狙い撃ち攻撃
- **強力な攻撃**
  - チャージ時間を設けることでプレイヤーへ攻撃の予兆を与え、発射時には密度の高い弾攻撃を行う。

- プレイヤーを狙う攻撃を行うことで、避けを意識させる役割があります。

## 出現処理（pop）

敵オブジェクトを有効化し、初期状態を設定する処理です。  
親クラス `Enemy` の `pop()` 関数を共通の出現処理として使用します。

### 実行内容

- **有効化**
  - `mIsActive` を `true` に設定し、更新（`action()`）および描画（`draw()`）の対象とする。
- **撃破状態のリセット**
  - `mIsDefeat` を `false` に設定する。
- **座標・個体番号の設定**
  - 引数で指定された座標（`popPositionX`、`popPositionY`）を設定する。
  - 個体判別番号（`mNumber`）を設定する。

### ステージ別の初期化

| 設定項目 | 通常ステージ（`isBossStage = false`） | ボスステージ（`isBossStage = true`） |
|----------|---------------------------------------|--------------------------------------|
| 初期ライフ | `ENEMY_MAX_HP`（30） | `ENEMY_MAX_HP_BOSS_STAGE`（60） |
| 開始カウント | `0`（コンストラクタ等の初期値） | `BOSS_STAGE_START_COUNT`（10） |
| 対象フラグ | `mIsBossStage = false` | `mIsBossStage = true` |

### 初期化の補足

ボスステージでは、召喚直後からボスの攻撃パターンと同期させる必要があるため、`mMoveCount` および `mShotCount` を `BOSS_STAGE_START_COUNT`（10）から開始することで、初期動作のタイミングを調整している。

##  移動仕様

### 移動パターン

| ステージ | フェーズ | 移動内容 |
|----------|----------|----------|
| 通常ステージ | 放物線移動 | 左右から出現し、画面中央（`SCREEN_CENTER_X`）を境に y軸の移動ベクトルに乗算している定数を を切り替え、放物線を描いて出現場所の反対側に移動する。 |
| ボスステージ | 1. 登場移動 | 召喚位置から指定待機位置まで `atan2` を利用したベクトル移動を行う。 |
| | 2. ワープ | チャージ完了後、`mIsWarp` を `true` にして画面端へワープする。 |
| | 3. 円運動 | `ARC_MOVE_START_FRAME`（331f）から `mArcMotionAngle` を更新し、中心点を軸とした円運動を開始する。 |

#### 放物線移動
  - 2フレームに1回移動処理を実行します。
  - 画面中央（`SCREEN_CENTER_X`）（`x=320`）に到達するまでは、y軸速度（`mVectorY`）を減速（`ENEMY_SLOWDOWN_Y `を乗算）させながら降下させます。中央を過ぎると、逆に速度を加速（`ENEMY_SPEEDUP_Y` を乗算）させながら上昇に転じます
  - 左右の出現位置でy軸速度（`mVectorY`）に乗算する定数に調整用のマージン（`CHARGE_ENEMY_SPD_MARGIN`）を加減算しています、
  - 終了条件：画面外（上下左右いずれかの端）に完全に到達した時点で終了。
  - 自身の座標 (mX, mY) に速度ベクトル（mVectorX,mVectorY）に加算して位置を更新しています。

#### 登場移動

出現位置から指定位置まで移動します。
| 項目  | 内容 |
| ---- | ---- |
| 目標座標 | （`CHARGE_ENEMY_SUMMON_POINT_LEFT_X`, `CHARGE_ENEMY_SUMMON_POINT_LEFT_Y`） = (160, 100) |
| 終了条件| 指定位置と現在の座標のずれを計算し、一定値(`ENEMY_POSITION_DIFFERENCE_RANGE`)内に収まる|

- **指定位置への移動ロジック**
  - 指定位置への移動には atan2() を用いて移動角度を算出する
  - 算出した角度から cos()・sin() を利用して速度ベクトルを求める
  - 指定位置と現在の座標のずれを計算し、一定値(`ENEMY_POSITION_DIFFERENCE_RANGE`)内に収まると停止する。

#### ボスステージ：ワープ移動仕様

ボスステージにおいて、チャージ攻撃完了後から円運動攻撃へ移行するための移動です。

#### 1. 実行トリガーと期間

| 項目 | 内容 |
|------|------|
| ワープ期間 | 出現から **280～330フレーム**（`WARP_START_FRAME` ～ `WARP_END_FRAME`）の間実行する。 |
| 状態管理 | ワープ中は `mIsWarp` を `true` に設定する。 |

---

### 2. ワープ演出と挙動

ワープ中は「画面内から消失し、別の場所へ出現する」過程を、移動・描画・エフェクトによって表現する。

#### 上昇演出

- ワープ開始時（280f）に `mVectorY` を上向きに設定する。
- フレーム経過に応じてY方向へ加速させる。
- X座標は固定し、Y座標のみを更新することで、上方へ吸い込まれるような演出を行う。

#### 透過度制御

- `mMoveCount` の進行に合わせて `mAlpha` を徐々に減少させる。
- フェードアウトすることで、敵が消失する演出を行う。

#### エフェクト再生

- `WARP_TRIGGER_FRAME`（304f）でワープエフェクト（`WARP_EF`）を再生する。
- 転送の瞬間を視覚的に強調する。

#### 描画切り替え

- `mIsWarp == true` の間は通常画像ではなくダメージ画像（青色）を表示する。
- プレイヤーがワープ状態であることを認識しやすくする。

---

### 3. ワープ終了処理

#### 座標更新

- `WARP_END_FRAME`（330f）に到達した時点でY方向の移動を停止する。
- `mX`、`mY` を円運動開始位置へ即座に更新し、円運動フェーズへ移行する。

#### 状態復帰

- `mIsWarp` を `false` に戻す。
- `mAlpha` を `MAX_ALPHA` に戻し、通常描画へ復帰する。

---

### 4. 設計意図

#### 無敵状態との連携

- チャージ中からワープ開始までの一定時間は `mIsInvincible` を有効にし、チャージからワープへの移行がプレイヤーの攻撃によって中断されないようにしている。

#### 視覚的フィードバック

- 単純な座標変更ではなく、上昇・フェードアウト・ワープエフェクトを組み合わせることで、空間を移動していることをプレイヤーへ分かりやすく伝える。

## 4. 攻撃仕様

### 攻撃設計意図

- **チャージ演出**
  - 発射前にチャージエフェクトを表示し、プレイヤーへ攻撃の予兆を与える。
- **無敵時間の付与**
  - チャージ中から攻撃開始まで（240f～280f）は `mIsInvincible` を有効にし、攻撃フェーズへ確実に移行できるようにする。

### 弾発射処理

| ステージ | 攻撃パターン | 実装内容 |
|----------|--------------|----------|
| 通常ステージ | プレイヤー狙い撃ち | チャージエフェクト（`CHARGE_EF`）表示後、プレイヤーを狙ってミニ弾を連続発射する。 |
| ボスステージ | 扇状弾幕 + 環状弾幕 | ・3方向へのミニ弾を18回連続発射<br>・10方向の環状通常弾（`CHARGE_ENEMY_RING_BULLET_AMOUNT`）を3回発射 |

---

## 5. ダメージ処理（継承）

- 親クラス `Enemy` の `takeDamage()` をそのまま利用する。
- 実行トリガーは `mIsTakeDamage` フラグによる判定。
- ボスステージでは `UNBEATABLE_START_FRAME` ～ `UNBEATABLE_END_FRAME` の間は無敵状態となり、ダメージを受け付けない。

- ワープ中（`mIsWarp`）はダメージ画像を表示するなど、視覚的な変化を伴う。

## 7. 補足・テクニカル事項

### 救済措置

- ボスステージでは攻撃開始時の特定フレーム
  （`CHARGE_ENEMY_BOSS_ITEM_DROP_FRAME`：345f）
  に無敵アイテム（`OBJECT_STAR`）をドロップする。

### 引数管理・シーケンス管理

- `EnemyActionData` 構造体によって必要なデータを一括で受け渡す。
- `mMoveCount` により行動シーケンスを管理する。
  - 例：331f以降に円運動へ切り替える。

### メモリ管理

- クラスメンバは基本変数のみで構成されるため、デストラクタで個別にメモリ解放を行う必要はない。
