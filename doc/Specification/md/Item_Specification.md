# Item(アイテム)仕様書
## 概要と設計意図
### 役割
- 敵を倒すとドロップするアイテムオブジェクト。プレイヤーに様々な効果を与え、ゲームプレイを補助するものです。
- プレイヤー機のパラメータに様々な影響を与えます
### 設計意図
- 移動、当たり判定等の機能を自己で行い、オブジェクト指向で作成します。
- 当たり判定を自己で行い、判定処理とアイテム効果反映をアイテム側とプレイヤー側で分ける目的で作成しています。
- プレイヤーへ自動で吸い付く機能を自己で判断し行い、プレイヤーが回収しやすいように作成しています。

## 基本パラメータ
アイテムには3種類あり、それぞれ別の効果をプレイヤーに与えます。
効果処理自体はプレイヤー側で行います。
- [プレイヤー : Player_Specification.md](Player_Specification.md)
### 初期位置
アイテム管理クラスの出現関数によって設定されます。
### 種類
アイテムは経験値、回復、無敵の効果を持つ3種類に分かれています。
アイテムの種類は`mItemObjectType`によって判別しています。
| 種類 | 画像 | 効果 |仕様定数|
| ---- | ---- | ---- | ---- |
|経験値|青い十字型|プレイヤーのショットパワーを増やす|`OBJECT_EXP`|
|無敵|黄色の星型|プレイヤーを数秒間無敵状態にする|`OBJECT_STAR`|
|回復|青いハート型|プレイヤーのHPを回復する|`OBJECT_LIFE`|

### 当たり判定サイズ
アイテムオブジェクトのサイズは、すべての種類（経験値・回復・無敵）で共通の定数が使用されています
|項目|値|仕様定数|
|----|----|----|
|横幅|20|`ITEM_OBJECT_WIDTH`|
|縦幅|20|`ITEM_OBJECT_HEIGHT`|

### アイテム効果
各アイテムの効果詳細です。効果の反映処理はプレイヤー側で行います。
#### 1.経験値アイテム
経験値アイテムはプレイヤーのショットパワーを上げることができます。
- **スコア加算**
   - 獲得時、現在のスコアに`EXP_SCORE`(50)点を加算します。
- **ショットパワー加算**
   - プレイヤーのショットパワーに`SHOT_POWER_INCREASE_AMONT`(30)増加させます。
   - ショットパワーによってプレイヤーの攻撃が段階的に強化されます。

#### 2.無敵アイテム
無敵アイテムはプレイヤーを数秒間無敵にすることができます。
- **スコア加算**
   - 獲得時、現在のスコアに`STAR_SCORE`(100)点を加算します。
- **無敵状態の有効化**
   - プレイヤーの無敵フラグ（`mUnbeatable`）が true になり、敵の攻撃によるダメージを一切受けなくなります
- **効果時間の延長**
   - すでに無敵状態であるときに取得した場合、残り時間は最大値まで回復（リセット）されます

#### 3.回復アイテム
回復アイテムはプレイヤーのHPを回復することができます。
- **スコア加算**
   - 獲得時、現在のスコアに`LIFE_SCORE`(200)点を加算します。
- **体力（ライフ）の加算**
   - プレイヤーの現在ライフ（`mLife`）が 1つ増加します
   - ライフがすでに最大値（`MAX_LIFE`: 3）に達している場合は、それ以上増えません

## 生成
アイテムは敵を倒すと出現します。種類によって出現条件が違います。
管理クラスによって生成は管理されています。

## 更新処理
アイテムの更新処理（action）では、移動、当たり判定、およびプレイヤーへの吸い付き（ホーミング）判定を毎フレーム行います

### 移動仕様
アイテムの種類に応じて、以下の2パターンの移動挙動を持ちます。
#### 1.経験値アイテムの移動
敵を撃破した際、放出されるような挙動から追従へと移行します。
- **拡散移動（初期挙動）**
  - 出現の瞬間、`OBJECT_EXP_FAST_SPEED` (8.0) の初速で6方向へ移動（拡散）を開始します.
  - 拡散の方向と移動ベクトルは出現時に管理クラスによって設定されます。
- **減速処理:**
  - `mEXPMoveCount` 変数を用いて移動時間を管理し、`OBJECT_EXP_SPEEDDOWN` (0.9f) を乗算することで徐々に減速します

- **吸い付き移動（ホーミング）**
  - 出現から `OBJECT_EXP_HOMING_START_FRAME` (30フレーム) が経過すると、プレイヤーへの吸い付き移動が開始されます

#### 2.回復・無敵アイテムの移動
画面内を漂いながら、プレイヤーの接近を待ちます。
- **降下移動**
  - 出現後、基本速度 `OBJECT_VECTOR_Y` (1.0f) でゆっくりと画面下部へ向かって移動します
- **吸い付き移動（ホーミング）**
  - 当たり判定よりも大きな判定で吸い付き移動の判定を行っています。
  - 吸い付きフラグ`mIsActiveHomingMove`がtrueになるとレイヤーへの吸い付き移動が開始されます

#### 吸い付き移動（ホーミング）について
- **有効化条件**
  - プレイヤーの中心座標との距離（X, Yそれぞれの絶対値）が `OBJECT_HOMING_RANGE` (100px) 以内に入ると、吸い付きフラグ`mIsActiveHomingMove`が true になります
  - 経験値アイテムのみ、時間経過で吸い付き移動へ移行します。

- **移動速度**
  - 吸い付き開始後は、移動速度が変更され、`OBJECT_ITEM_HOMING_SPEED` (10.0f) の速度でプレイヤーに向かって直線的に移動します。

### 当たり判定仕様
アイテムの当たり判定は、矩形同士の重なりではなく、プレイヤーとの距離に基づいたロジックで処理されています。
- **判定方法**
   - アイテムとプレイヤーの各中心座標の距離（差分の絶対値）を計算し、一定距離内に近づいた場合に「当たった」と判定します
- **実行内容**
   - 判定が行われるとプレイヤーへ当たったアイテム種類を渡し、プレイヤーのアイテムヒットフラグ`mIsItemHit`をtrueにします。
