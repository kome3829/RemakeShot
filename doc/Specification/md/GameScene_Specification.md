# ゲームシーン

## シーン概要

ゲームの本編(メインループ)を管理し、プレイヤーの操作、敵の挙動
ステージの進行などを制御する。

## 設計意図
各種クラス（プレイヤー、敵管理、弾管理、アイテム管理等）を一括管理し、
ゲーム進行処理を行う。
---
## 基本的なゲームシーンフロー

```text
ゲーム開始
    ↓
通常ステージ（雑魚敵）
    ↓
全敵撃破
    ↓
ボス登場演出
    ↓
ボスステージ
    ↓
ボス撃破
    ↓
クリア演出
    ↓
リザルトシーン

```

## 表示内容
ゲームシーンで表示する要素は以下の通りです。
| 表示要素 | 内容 | 詳細・補足 |
|:---------|:-----|:-----------|
| プレイヤー | 自機（青色） | ダメージ時は赤色、無敵時は黄色に変化。無敵中は点滅演出を行う。 |
| 敵 | 雑魚敵・ボス | 被弾時はダメージ用画像（雑魚敵：青色、ボス：白）へ切り替わる。 |
| プレイヤー弾 | 通常・ミサイル・スペシャル | ショットパワーに応じて弾の種類と発射数が増加する。 |
| 敵弾 | 通常弾・ミニ弾 | ボス敵とチャージ敵で使用するミニ弾の色が違う |
| エフェクト | 爆破・チャージ・ワープなど | ボス撃破時は20連続の爆発エフェクトを再生する。 |
| プレイヤーUI | HPアイコン・無敵ゲージ | 自機下部にハート型のHPアイコンと、無敵残量を示すオレンジ色のゲージを表示する。 |
| ボスHPバー | 残りHPの割合表示 | ボスステージ中のみ、画面上部に赤色のHPバーを表示する。 |
| 下部UIエリア | スコア・タイム・パワー | 画面下部に各数値情報をまとめて表示する。 |
| 特殊演出 | ボス警告・ゲームオーバー ・ゲームクリア| 画面全体のフェードやテロップ表示などの演出を行う。 |
| 背景 | ステージ背景動画 | 約600フレーム（約10秒）ごとにループ再生される。 |

### UI表示詳細
下部UIやボスHPバーなどはゲームシーンで描画処理を行います。

#### ボスHPバー
1. 表示条件
   - `mStageType` が `BOSS_STAGE`（ボスステージ） の間のみ表示
2. 役割
   - ボスの最大HPに対する現在の残り体力を、横棒グラフの形式でリアルタイムに表示します
3. 計算ロジック
   - バーの長さはボスの残りHPの割合に基づいて算出されます。
   - バーの右端座標計算: 最大幅 315px (`BOSS_HP_BER_MAXWIDTH`) に対し、上記のHPパーセントを掛け合わせることで、描画すべきバーの右端座標（`mBossHitPointBarRightEnd`）を決定します
4. 外観と配置
   - HPバーは、外枠となる矩形と、その内側に表示される体力ゲージの2層構造で構成されています。

| 項目 | 使用定数 | 値（座標・サイズ） |
|:-----|:---------|:------------------:|
| 表示位置（外枠左上） | `BOSS_HP_BAR_LEFT_X`<br>`BOSS_HP_BAR_TOP_Y` | (0, 0) |
| 表示範囲（外枠右下） | `BOSS_HP_BAR_RIGHT_X`<br>`BOSS_HP_BAR_BOTTOM_Y` | (320, 14) |
| 内側ゲージの開始位置 | `BOSS_HP_INNER_LEFT_X`<br>`BOSS_HP_INNER_TOP_Y` | (2, 2) |
| 内側ゲージの下端 | `BOSS_HP_INNER_BOTTOM_Y` | 12 |
| ゲージの最大横幅 | `BOSS_HP_BER_MAXWIDTH` | 315px |
| テキスト表示位置 | `BOSS_HP_TEXT_X`<br>`BOSS_HP_TEXT_Y` | (10, 20) |

5. 描画の優先順位
   - 背景動画、キャラクター（プレイヤー・敵）、弾よりも後の最前面に描画されます

## 開始時処理

- タイトルBGMを停止する
- ゲームBGMを再生する
- 各オブジェクト（プレイヤー、敵など）の初期化関数の実行

---

## 更新処理

- 各種マネージャークラス（雑魚敵・エフェクト・アイテム・弾）の更新処理の実行
- ステージタイプ（NOMAL, BOSS, CLEAR, TEST）に応じた進行管理
- 背景動画のカウント更新とループ位置の制御
- プレイヤー・ボスの更新処理の実行
- 当たり判定:プレイヤー弾 vs 敵、敵弾 vs プレイヤー、アイテム vs プレイヤー
- プレイヤー・敵の撃破判定
- ゲーム内経過時間の更新

### 各ステージでの処理内容
- NOMAL_STAGE:雑魚敵の出現関数を実行。ボス出現フラグ(mIsBossAlert)によって、ボス出現演出処理を実行している
- BOSS_STAGE:ボス撃破フラグ(mIsDefeat)によって、ボス撃破演出を実行している。
- CLEAR_STAGE:リザルトシーン切り替え処理を行う。
- TEST_STAGE:テスト用のパラメータ設定、ワープエフェクトの再生。ボスステージへ遷移を行う。

## ボスイベント
### ボス出現演出
 **実行条件**
- 通常ステージの雑魚敵が全滅、または画面外へ退場した後に実行されます。
- 通常ステージにおいてボス出現演出フラグ（`mIsBossAlert`）が true になると実行されます
- 詳しくは敵管理クラスの仕様で説明する。

 **演出内容**
 - 画面全体を覆う真っ赤な画像をアルファブレンド（透過）させながら表示し、素早いフェードアウトを繰り返します。

**制御**
- フェード速度：`FADE_OUT_SPEED`(3)の速さで毎フレームと透過度を下げていきます。アルファ値が0以下になると最大値ヘリセットします。
- 繰り返し回数：`REDBG_FADEOUT_COUNT_MAX` (3) 回フェードアウトを繰り返すことで演出が完了します

### ボスの出現処理
ボス出現演出が終了時に実行されます。
- [ボス敵 : BossEnemy_Specification.md](BossEnemy_Specification.md)

 **ステージ切り替え**
- 現在のステージ状態を示す`mStageType `を`NOMAL_STAGE `から
`BOSS_STAGE`に切り替えます。

 **ボスオブジェクトの有効化:**
- ボス敵の有効化フラグ（mActive）を true に設定し、毎フレームの更新（action）と描画（draw）を開始します
### ボス撃破演出
- 一定回数のランダムな位置で爆破エフェクトとSEを再生し、終了後クリア処理に移行します。

- ボスのHPが0（BOSS_HP_PER_DEFEAT）に達した際に実行される撃破演出です。

---
## 終了判定と遷移条件

- リザルトシーンへの遷移（勝利）： ボスを撃破し、撃破演出が完了したとき
- リザルトシーンへの遷移（敗北）： プレイヤーの撃破フラグが立ち、一定の遅延カウントが経過したとき
- ゲーム終了： Escapeキーが押されたとき
- データ保存： シーン遷移直前に、スコアやクリア時間をデータクラスへ保存する

## スコアシステム詳細

スコアは、プレイヤーの行動に応じて加算されるゲームプレイの成果値です。敵への攻撃や敵の撃破、アイテム取得によって増加し、ゲーム中はUIに表示されます。また、ゲーム終了時にはリザルト画面へ引き継がれます。

### スコア加算条件

| トリガー | 加算スコア | 使用定数 |
|----------|-----------:|----------|
| 敵に弾が命中 | 150 | `HIT_SCORE` |
| 敵を撃破 | 1000 | `ENEMY_SCORE` |
| 経験値アイテムを取得 | 50 | `EXP_SCORE` |
| 回復アイテムを取得 | 200 | `LIFE_SCORE` |
| 無敵アイテムを取得 | 100 | `STAR_SCORE` |
### UI表示仕様

| 項目 | 内容 |
|------|------|
| 表示位置 | 画面下部UIエリア左側 |
| X座標 | `UI_SCORE_X`（10） |
| Y座標 | `UI_BOX_TOP`（460） |

### データ管理

- スコアは `mScore` で管理する。
- ゲーム終了時に `Data` シングルトンクラスの `mResultGameScore` へ保存する。
- 保存したスコアはリザルト画面で使用する。

## ゲーム経過時間
ゲーム開始からの経過時間を計測し、ゲーム画面へ表示します。計測した時間はゲーム終了時にリザルト画面へ引き継がれます。

### 計測開始
- ゲームシーン開始時に `GetNowCount()` を使用して開始時刻を取得します。
- 取得した時刻は`mStartTime`の保存します。

### 計算方法
- 更新処理で毎フレーム `GetNowCount()` で現在時刻を取得します。
- 開始時刻`mStartTime`との差分から経過時間を算出しています。

### 時間変換

ゲームシーンのUI表示では秒だけで、リザルトシーンでは分と秒で表示しています。
| 項目 | 内容 | 使用定数 |
|------|------|----------|
| ミリ秒 → 秒 | 経過時間(ms) / 1000 | `TIME_MILLISECOND_TO_SECOND` |
| 秒 → 分・秒 | 秒数を60で割り、分と秒へ変換 | `TIME_SECOND_PER_MINUTE` |

### データ管理
シーン終了時に、分および秒として Data クラスへ保存され、リザルトシーンへ引き継がれます。
- 経過時間の「分」は `mInGameTimeMinutes` に保存します。
- 経過時間の「秒」は `mInGameTimeSecond` に保存します。

## 補足・テクニカル事項

### マジックナンバーの排除と定数管理
 - コーディング規約に基づき、ソースコード内でのマジックナンバーの使用は禁止されており、すべての数値には意味のある名前が付与されています
 - UI表示座標: スコア（UI_SCORE_X: 10）、残り時間（UI_TIME_X: 200）、ショットパワー（UI_SHOT_POWER_X: 350）、UIエリアの上下境界（UI_BOX_TOP: 460, UI_BOX_BOTTOM: 480）など、表示位置はすべてヘッダファイルで定義されています
 - ボスHPバーの矩形範囲（BOSS_HP_BAR_LEFT_X: 0, TOP_Y: 0, RIGHT_X: 320, BOTTOM_Y: 14 など）
 - 演出パラメータ: ボス出現時の背景フェード速度（FADE_OUT_SPEED: 3）や、フェードアウトを繰り返す回数（REDBG_FADEOUT_COUNT_MAX: 3）などが管理されています
### メモリ管理
 - インスタンスの生成: コンストラクタにおいて、各オブジェクトマネージャーのインスタンスを生成しています
 - 現状のデストラクタは空であり、インスタンスの解放処理(delete)の実装が今後の課題である
 - 生成するオブジェクト
   - Player
   - BossEnemy
   - BulletManager
   - MinionEnemyManager
   - EffectManager
   - ItemObjectManager
### マネージャークラスによる統一管理
- オブジェクトの生成・更新・描画・破棄のライフサイクルを各マネージャーに委譲することで、シーン本体のロジックを簡素化し、保守性を高めるため
### 背景動画の再生制御
- 背景動画は、プログラム側でループ位置を制御しています
- ループ仕様: 背景動画は600フレーム（約10秒）で1ループするよう設計されています
- カウント処理: action関数内で毎フレームmMovieCountを更新し、規定のフレーム数に達した際に再生位置を先頭（MOVIE_LOOP_START: 0）へ戻す処理を行っています
### 時間計算の仕様
- ゲーム内の経過時間はミリ秒単位で取得され、表示用に変換されます。
- 変換処理: 内部で保持しているミリ秒単位の時間を、秒（TIME_MILLISECOND_TO_SECOND: 1000）および分（TIME_SECOND_PER_MINUTE: 60）へ変換するための定数を用いて計算し、UIに反映します
### テスト・デバッグ用機能(TEST_STAGE)
- 確認のため、通常のゲームフローとは別にTEST_STAGE状態が用意されています
- テスト用パラメータ: この状態では、プレイヤーのショットパワーがテスト用（TEST_PLAYER_POWER: 6100）に底上げされ、無敵状態（mIsUnbeatable = true）に設定されます
- 挙動確認: Spaceキー押下によるワープエフェクトの表示確認や、ボス出現演出からボス戦への遷移確認を即座に行うことが可能です
### シーン遷移時のデータの引継ぎ
- リザルトシーンに遷移する前に、プレイ結果をデータクラスへ保存しています。
- 保存項目: mScore（最終スコア）およびmTime（クリア時間）が、シングルトンインスタンスであるDataクラスの変数へ書き込まれます
。これによって、別シーンでこれらの値を参照しています
